KO – GF PRE-POPULATE RADIO FIELDS VERSION 1.0.0 (PRODUCTION) AUTHOR: KO

********** OVERVIEW **********

KO – GF PRE-POPULATE RADIO FIELDS AUTOMATICALLY MIRRORS A SELECTED VALUE
FROM ONE GRAVITY FORMS RADIO FIELD (PARENT) INTO ANOTHER RADIO FIELD
(CHILD).

THIS IS DESIGNED FOR MULTI-PAGE GRAVITY FORMS WHERE USERS ARE ASKED TO
SELECT THE SAME METRIC OR OPTION MORE THAN ONCE (FOR EXAMPLE: MILES /
KILOMETERS).

THE PLUGIN SUPPORTS:

-   UNLIMITED RULE GROUPINGS
-   MULTI-PAGE FORMS (AJAX AND NON-AJAX)
-   CONDITIONAL LOGIC
-   FULL PAGE RELOAD NAVIGATION
-   SAFE VALUE PERSISTENCE USING SESSION STORAGE
-   NO JQUERY DEPENDENCY

********** HOW IT WORKS **********

WHEN A USER SELECTS A RADIO OPTION IN THE PARENT FIELD, THE PLUGIN:

1.  STORES THE SELECTED VALUE IMMEDIATELY
2.  PERSISTS THE VALUE ACROSS PAGE TRANSITIONS
3.  AUTOMATICALLY SELECTS THE MATCHING OPTION IN THE CHILD FIELD
4.  RE-APPLIES THE SELECTION IF THE CHILD FIELD LOADS LATER DUE TO
    CONDITIONAL LOGIC OR PAGE NAVIGATION

THE PARENT AND CHILD RADIO FIELDS MUST HAVE MATCHING LABELS AND/OR
VALUES.

********** INSTALLATION **********

1.  UPLOAD THE PLUGIN ZIP FILE VIA: WP ADMIN → PLUGINS → ADD NEW →
    UPLOAD PLUGIN

2.  ACTIVATE THE PLUGIN.

3.  NAVIGATE TO: WP ADMIN → SETTINGS → KO PRE-POPULATE RADIO BUTTONS

********** CONFIGURATION **********

ADD A RULE USING THE FOLLOWING:

FORM ID – THE GRAVITY FORM ID. PARENT FIELD ID – THE RADIO FIELD TO COPY
FROM. CHILD FIELD ID – THE RADIO FIELD TO POPULATE.

EXAMPLE:

FORM ID: 2 PARENT FIELD ID: 59 CHILD FIELD ID: 75

CLICK SAVE RULES.

THAT IS ALL THAT IS REQUIRED.

********** REQUIREMENTS **********

-   BOTH PARENT AND CHILD MUST BE RADIO BUTTON FIELDS.
-   LABELS AND/OR VALUES SHOULD MATCH (EXAMPLE: MILES / KILOMETERS).
-   DESIGNED FOR GRAVITY FORMS ENVIRONMENTS.

********** MULTI-PAGE BEHAVIOR **********

THIS PLUGIN HANDLES:

-   NON-AJAX MULTI-PAGE FORMS
-   AJAX-ENABLED FORMS
-   USERS IMMEDIATELY CLICKING NEXT
-   USERS NAVIGATING PREVIOUS → NEXT
-   CONDITIONAL LOGIC THAT INJECTS FIELDS LATER

IT USES EVENT DELEGATION, SUBMIT INTERCEPTION, MUTATION OBSERVERS, AND
SESSION STORAGE TO ENSURE RELIABLE POPULATION.

********** LIMITATIONS **********

-   DOES NOT MAP DIFFERENT VALUES (A → B). VALUES MUST MATCH.
-   DOES NOT FORCE-LOCK THE CHILD FIELD.
-   DOES NOT OVERRIDE A MANUAL USER CHANGE AFTER AUTO-POPULATION.

********** DEBUG MODE **********

INSIDE THE PLUGIN PHP FILE YOU WILL FIND:

‘debug’ => false,

CHANGE TO:

‘debug’ => true,

TO ENABLE CONSOLE LOGGING FOR TROUBLESHOOTING.
